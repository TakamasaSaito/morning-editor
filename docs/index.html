Google Cloud 82%増で圧勝、米関税期限・AI投資競争が同時進行

AI

Google Cloud 82%増、AI需要で全面突破

Alphabetが7月22日に発表したQ2 2026決算で、売上高は前年比24%増の1198億ドルと市場予測を上回った。Google Cloudが82%増の248億ドルと急成長し、エンタープライズAI需要が主因とされている。

AIインフラへの企業支出が想定を大幅に超えるペースで拡大していることが数値で裏付けられ、クラウド市場の競争地図が塗り替わりつつある。AlphabetはAIインフラ投資として2026年通期の設備投資を最大2050億ドルに引き上げており、業界全体の投資競争を加速させる可能性がある。

世界情勢

米関税Section 122、明日失効へ確度:高

トランプ政権が最高裁のIEEPA違憲判決後に発動した10%の一律輸入追加課税（Section 122）が、7月24日（明日）に期限切れとなる。議会が延長しない場合、代替措置としてSection 301による10〜12.5%の関税が46か国に適用される方向で準備が進んでいると複数の法律専門家が伝えている。

AI

MS・Mistral、欧州AI主権で提携拡大確度:高

MicrosoftとMistral AIは7月21日、規制産業・欧州市場向けにフロンティアAIを展開する戦略的提携の大幅拡張を発表した。MicrosoftがMistralの欧州データセンター拡充に数十億ドル規模の投資を行い、Mistralのモデルが「Microsoft Foundry」「Copilot Studio」「Azure」上で利用可能になる。

ビジネス

AIロビー支出、四半期で23%増確度:高

米連邦ロビー活動開示資料によると、主要AIラボのQ2 2026合計政治ロビー支出は前四半期比23%増の317万ドルに達した。Anthropicが197万ドル（前四半期比26%増）でOpenAI（120万ドル、同18%増）を大きく上回り、NVIDIAに並ぶ規模に迫っている。

日本

小泉防衛相「核議論タブーなし」確度:中

小泉進次郎防衛相は7月23日、核をめぐる議論について「タブーなく議論すべき」と発言したとNHKが報じた。国会会期末を前に「副首都」法案の採決調整も大詰めとなっており、安全保障・国内政治の両面で重要な動きが重なっている。

テクノロジー

オープンモデル週、7月末に集中確度:高

Moonshot AIが開発するKimi K3のオープンウェイト公開が7月27日に予定されており、DeepSeek V4の安定版リリース（7月24日）と時期が重なる。7月末の1週間が「業界史上最大のオープンウェイト集中リリース期間」になると複数メディアが伝えている。

背景

Cloudは2期連続加速、AI実需が確認された

Q1 2026のCloud成長率63%からQ2は82%へさらに加速した。Alphabetによると、Fortune 100企業の約90%がGemini Enterpriseを利用しており、Geminiモデルは毎分220億APIトークンを処理しているという。GAAP純利益は1121億ドルと前年比298%増だが、これは主に株式評価益などの一時的要因によるもので、コア営業利益の30%増（408億ドル）が実力値とみるべき構造だ。

展望

2050億ドル投資で供給制約に対応

Alphabetが設備投資上限を2050億ドルへ引き上げた理由として、CFOのアナット・アシュケナジー氏は「供給制約が続く中、需要拡大に対応するため」と説明している。7月24日には米国のSection 122関税が期限を迎えるため、AI半導体・サーバーの調達コストが変動するリスクも残る。

影響

Gemini Enterprise普及が戦略立案を急かす

エンタープライズAIの採用がFortune 100レベルで急速に進んでいる現実は、IT戦略の立案スケジュールを前倒しする圧力になりうる。AIインフラのマルチクラウド戦略や、Gemini Enterprise採用の競合他社動向は、早期に把握しておく価値がある可能性がある。

  • 自社のクラウド戦略ロードマップにおけるGoogle Cloud / Gemini Enterprise採用の競合状況を今週中に確認する
  • AIインフラ調達計画に関税リスク（7月24日Section 122失効）の影響シナリオを追加しておく

米関税Section 122、明日失効へ
背景

IEEPA違憲後、Section 301へ移行中

Section 122（1974年通商法）は150日間のみ有効な「緊急輸入課税」で、議会延長がなければ失効する。米国際貿易裁判所（CIT）は5月7日に同課税を違法と判断したが、政府は上訴中で現在も徴収継続中という複雑な状況にある。USTR（米通商代表部）はSection 301調査をすでに完了しており、新たな課税の法的根拠を準備済みとみられる。

展望

恒久関税化で貿易構造が変質する

7月31日には製薬品オンショアリング期限、9月1日にはEU航空機部品の関税変更と、今後も関税スケジュールが続く。Section 301への移行が実現すれば、貿易政策の枠組みがより恒久的な構造に変わる転換点となりうる。

影響

ITインフラ調達コストに直接波及しうる

エンタープライズ向けITインフラ（サーバー・ネットワーク機器）の調達コストが関税変動に連動する可能性があり、調達計画の見直しが必要になるかもしれない。IT戦略策定において、サプライチェーンの地政学リスクを評価するタイミングが来ている可能性がある。

  • 調達部門と連携し、主要IT機器の輸入関税コストへの影響シナリオを明日中に確認する
  • Section 301の対象国リスト（日本・韓国・台湾含む）と自社調達ルートの重複を点検しておく

MS・Mistral、欧州AI主権で提携拡大
背景

欧州規制産業向けソブリンAI基盤が形成

MicrosoftはNVIDIA Vera Rubin GPUを使ったMistralの欧州データセンター建設を支援し、クラウドから完全切断（エアギャップ）環境まで幅広い展開オプションを提供すると両社は説明している。Mistralはブルームバーグ報道によると約30億ユーロ（約204億ユーロ評価額）の資金調達を検討中とされるが、CEOはコメントを控えている。

展望

米欧融合モデルがアジアにも波及しうる

欧州AI規制と米国AI企業の地政学的緊張が続く中、「米欧テクノロジーの融合」モデルが一つの解として定着するか注目される。MicrosoftはSovereign Cloud戦略をアジア・中東にも拡張しており、類似の提携が他地域でも進む可能性がある。

影響

AIインフラ設計に「データ主権」の視点が必須に

規制準拠を重視するエンタープライズAI導入の設計指針として、「ソブリンAI」アーキテクチャのパターンが参照されうる局面が来る可能性がある。自社IT戦略にAIワークロードの配置主権（どの国のインフラで動かすか）の観点を組み込む必要性が高まるかもしれない。

  • Azure Sovereign Cloud / Mistral連携の技術仕様を確認し、自社のデータ主権要件に照合する
  • 日本の規制要件（医療・金融・政府系）との適合性を評価する観点でベンダーへのヒアリングを計画する

AIロビー支出、四半期で23%増
背景

AIラボの政治工作が急速に本格化

ロビー活動の重点課題として、サイバーセキュリティ、著作権、クラウドコンピューティング、防衛調達が挙げられている。テック最大手のMetaは599万ドルと依然最大のロビー支出者だが、前四半期比15%減と縮小傾向にある。

展望

規制環境がロビーで左右される局面へ

フロンティアAI公開前審査の法制化議論（既報）や、Section 301関税でのAI半導体扱いなど、政策の焦点が多岐にわたる中で、AIラボのロビー支出はさらに増加する可能性が高い。

影響

AI調達にベンダーの政治リスク評価が必要に

政府系・金融系の大規模AI調達においては、ベンダーの規制対応姿勢や政治的立ち位置がリスク要因として浮上する可能性がある。AI関連法制の動向モニタリングを調達・戦略部門の定常業務に組み込む必要性が高まるかもしれない。

  • 主要AIベンダーのロビー活動テーマ（著作権・サイバー・防衛）と自社IT戦略の交差点を四半期ごとにレビューする仕組みを検討する
  • 政府系プロジェクトに関わる場合は、AI調達基準の政策変化をウォッチするリスト（法案・NIST文書等）を整備する

小泉防衛相「核議論タブーなし」
背景

防衛政策が安保強化の新局面に入った

発言は国内の防衛費増額・防衛力整備計画の延長線上にあり、北朝鮮の核戦力強化や中国の軍備増強を背景とした安全保障議論の一環とみられる。「副首都」法案は国会会期末の採決調整が最終段階に入っており、国土強靭化・BCPの文脈でも注目される。

展望

会期末に向け安保法制が加速する可能性

今国会会期末（通常7月末〜8月）にかけて、防衛・安保関連の法整備が加速する可能性がある。核議論が具体的な政策提案につながるかは現時点で不明で、今後の政府の公式見解が焦点となる。

影響

公共系ITの要件変化に早期対応が求められうる

政府のBCP・国家安全保障戦略の動向は、企業のレジリエンス要件や政府系ITシステムの要件定義に影響しうる。防衛・公共分野のエンタープライズITに関わる場合は、政策変化の速度が上がる可能性を念頭に置いておく価値があるかもしれない。

  • 防衛・公共ITプロジェクトに関わる場合は、今国会会期末の法案成立動向を週次でウォッチする
  • 国家安全保障戦略の改定や防衛費増額の具体的内容が自社IT戦略（セキュリティ・BCP要件）に与える影響を確認する

背景

7月末、オープンウェイトモデルが一斉登場

Kimi K3は7月16日にAPIとして先行公開済みで、オープンウェイト版の公開によりオンプレミス・エアギャップ環境での利用が可能になる。DeepSeek V4はV3からの安定版アップデートと位置付けられており、中国発モデルの品質向上が続いている。

展望

モデル競争が閉鎖型/開放型の両軸で激化

オープンモデルの性能向上が商用モデルに追いつく速度が加速しており、AIモデル選定の競争環境が一段と激しくなると見られる。エンタープライズ向けにはモデル切り替え可能なアーキテクチャ（ルーティング層）の価値がさらに高まる可能性がある。

影響

社内AI評価体制の有無が戦略差別化に直結

自社でAIモデルを運用・評価する体制が整っているかどうかが、今後の調達判断の速度に影響しうる。オープンウェイトモデルの評価・PoC能力をIT部門が持てるか否かが競争力の差になるかもしれない。

  • Kimi K3・DeepSeek V4のベンチマーク結果が出次第、自社ユースケースへの適用可能性をPoCチームに事前確認する
  • オープンウェイトモデルの社内運用に必要なインフラ要件（GPU、セキュリティ、ライセンス）を事前に整理しておく
