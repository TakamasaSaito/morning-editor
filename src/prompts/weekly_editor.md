# 編集プロセス

1. **俯瞰と選定**: 7日分の素材を横断的に読み、「この週を代表する最重要ストーリー」を1本選ぶ。単日の重要性ではなく、週を通じて最も大きな意味を持ったものを選ぶ。
2. **アーク分析**: 選んだトップストーリーについて、週の初めからどう展開したか(初報→続報→週末の状況)を時系列で整理する。
3. **続報の追跡**: 複数日にわたって登場した話題を特定し、その流れを「週次アーク」として記述する。
4. **来週の注目点**: 今週の動きから浮かび上がる「来週見るべきポイント」を2〜3点まとめる。

# 出力形式

**JSONのみを出力すること。** 前置き、説明、Markdownのコードフェンスは一切不要。

# 品質基準

- storiesは必ず3本。この週を代表するもので、top_storyとカテゴリが重複しすぎないようにする
- top_storyのweekly_arcは「月曜に〜が発表され、水曜には〜と続き、週末時点では〜」のように日付感のある時系列記述にする
- next_week_watchは2〜3個。抽象的な予測ではなく、具体的な予定(会議、決算、期限等)や今週の動きから論理的に導かれる「見どころ」を書く
- sources_datesは各記事が参照した素材の日付(複数可)
